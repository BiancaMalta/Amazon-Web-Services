## O que é Cloud Computing ?
Antigamente, as empresas precisavam ter uma infraestrutura físia para armazenar seus dados, o que dava escopo para inúmeros problemas. Se aproveitando dessa dor, a Amazon, por posuirem uma enorme capacidade computacional, começou a oferecer seu serviço. No inicio cada empresa podia “alugar” espaço e poder computacional, pagando pelo seu armazenamento. Iniciativa esta que, revolucionou o mundo da tecnologia, tornando a AWS referencia em gestão de informação digital. Hoje, ela conta com mais de um milhão de clientes em 190 países e oferece mais de 217 serviços diferentes. Para ajudar a compreender alguns desses serviços, darei uma breve visão geral.

## Vantagens
<img align="right" src="https://github.com/BiancaMalta/Cybersecurity/assets/92928037/cc029d0d-87d9-488a-9749-b16e71a00020" width="37%">
Pioneira em Cloud Computing, sua infraestrtura e gerenciamento possuem cinco pilares:

- **Segurança/ Prevenção de perdas:**</br>
A AWS possui um sistema complexo para garantir tanto a segurança fisica do equipamento, quanto a virtual dos dados, são diversos recursos de proteção.
- **Eficiência de performance/ Escalabilidade:**</br>
O sistema possui atualizações autómaticas e periódicas,somado a isso, a solução implementada pode crescer junto com a sua empresa.
- **Confiabilidade/ Controle de qualidade:**</br>
Dados centralizados e sob os mesmos padrões
- **Excelência operacional:**</br>
Disponíveis a partir de qualquer dispositivo, global em minutos.
- **Otimização de custos/ Praticidade:**</br>
Não é necessário adquirir, instalar e manter um infraestrutura física. </br>Além disso, é possível aumentar ou diminuir os recursos, soluções sob demanda.

## Tipo de Cloud
### A respeito dos tipos de cloud, o leque de opções disponibilizadas é grande, focaremos nossa atenção em três: 

<img  align="left" src="https://kinsta.com/wp-content/uploads/2020/12/Cloud-Delivery-Models.png" width="55%"> 

- **IaaS** </br> 
Infraestrutura como serviço, como máquinas virtuais, servidores, etc. -> AWS EC2
- **PaaS** </br> 
Plataforma como serviço, nao precisa se preocupar com o tipo de banco, se é uma VM. -> AWS Beanstalk
- **SaaS** </br> 
Software como serviço, produto mais granular, permite acesso a um software sem instalar. -> Google Docs, Office 365

## Deployment models
- **Private cloud** </br> 
 Seu cloud, privado, aluguel de servidor físico, alto custo.
- **Public Cloud** </br> 
Qualquer um pode usar, o provedor de cloud oferece esse serviço ao público geral.
- **Hybrid Cloud** </br> 
A junção das duas na sua empresa.
- **Multicloud** </br> 
 Pode rodar em qualuqer cloud.
- **Community** </br> 
 Empresas se reunem e fazem um cloud.

## Serviços:
| Serviços | Função |
|---|---|
| VPC (Virtual Private Cloud) | Usúario inicia recursos em uma rede definida pelo mesmo |
| VPC Peering |  Conexão de rede entre duas VPCs/ compartilhamento de recursos, arquivos ou dados |
| Roles | Políticas de permissões que determinam o que pode ser feito ou não |
| NAT | Instâncias em uma sub-rede privada se conectem a internet ou a outros serviços |
| Security Gateways | Comunicação entre a VPC e a internet |
| Routes | Regras para que lugar o tráfego de rede de uma sub-rede ou gateway é direcionado |
| CloudWatch | Monitorar os recursos e aplicativos executados na AWS |
| EC2 (Amazon Elastic Compute Cloud) | O serviço que oferece os servidores virtuais da AWS |
| S3 (Amazon Simple Storage Service) | Armazenamento para a internet |
| Lambda | Executar o código sem precisar gerenciar e provisionar um servidor |
